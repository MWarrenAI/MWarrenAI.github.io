The Technical Challenge

The core challenge was implementing a Global State-Switching Mechanic within a 24-hour sprint. I engineered an event-driven system where a single "Pulse" triggers state changes across all environmental "Light Listeners," allowing for complex puzzle logic without heavy per-frame overhead.

State-Driven Architecture

Used a centralized C# Manager to broadcast state changes. This decoupled the light logic from individual NPC scripts, ensuring 60FPS even with multiple active elements.

Pipeline Management

Coordinated 11 unique character assets. When repository naming conflicts arose mid-jam, I enforced a new directory standard to ensure the final build remained stable.
